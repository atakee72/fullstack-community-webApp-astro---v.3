# Kiez dashboard notes

Loaded lazily when Claude reads/edits files in `src/components/kiez/`. Covers both the kiosk Svelte frontend and the data pipeline (sync script + APIs) — the section cross-references `scripts/sync-stats.ts` and `src/pages/api/kiez-stats.ts`. The legacy dashboard (`KiezDashboard.svelte`, carousels + IntersectionObserver reveal) was deleted in the kiosk rebuild (Tasks 1–11, July 2026); `src/components/kiez/plrPaths.ts` is the one legacy file kept (consumed by `KzMap`).

### Data pipeline (unchanged by the kiosk rebuild)
- **Data pipeline**: `scripts/sync-stats.ts` downloads XLSX from AfS (demographics) and MSS (social index), parses with `exceljs`, upserts to MongoDB
- **Sync schedule**: GitHub Actions workflow runs 2x/year (March + September) + manual dispatch
- **LOR codes**: Schillerkiez = 4 Planungsräume (2021+ LOR): `08100102` (Schillerpromenade Nord), `08100103` (Süd), `08100104` (Wartheplatz), `08100105` (Silbersteinstraße). Pre-2021: 2 PLRs `08010117` (Schillerpromenade) + `08010118` (Silbersteinstraße). Sync script auto-detects based on period.
- **MSS column layout** (corrected 2026-07-14, verified against both 2023 files' Erläuterungen sheets): SAME column order in all report years — S1=unemployment(col4), S3=transfer(col6), S4=child-poverty(col7). Col5 (S2) is pre-2023 Langzeitarbeitslose (not stored) / 2023+ share of children in single-parent households, stored as `single_parent_children_rate` (2023+ only — NOT a poverty measure). The former "2023+ layout" branch (CP=5, TR=7) was a misreading that swapped Kinderarmut and Transferbezug for period 2023; `youth_unemployment_rate` never held real youth-unemployment data and was removed.
- **API**: `GET /api/kiez-stats` — public, no auth, 24h cache. Aggregates demographics + social data, pre-computes non-overlapping migration segments (MH includes AUSL)
- **Air quality**: Live data from BLUME API station MC042 (Nansenstraße). `GET /api/kiez-air` proxies LQI grades (1–5) with 30 min cache. No auth, no MongoDB, no sync script. Cards show German pollutant names (Feinstaub, Stickstoffdioxid, Ozon, Kohlenmonoxid) with abbreviations below, plus a color-coded grade scale legend. Pollutants with no current reading show "keine Angabe" instead of being hidden.
- **Historical backfill (demographics)**: `bash scripts/backfill-history.sh` (one-time, 6 periods). Use `--dry-run` first.
- **Historical backfill (social)**: `bash scripts/backfill-social.sh` (one-time, 5 MSS periods 2013–2021). Use `--dry-run` first.
- **Dry-run**: `pnpm tsx scripts/sync-stats.ts --dry-run` — parses XLSX without DB writes (for verifying structure)
- **Air logger (Messwert-Logger, since July 2026)**: GitHub Actions (`.github/workflows/kiez-air-logger.yml`, `7,37 * * * *`) curls `GET /api/cron/log-air` (Bearer `CRON_SECRET`, fail-closed 503 when unset — same gate as process-deletions). Tick = fetch BLUME → upsert into `schillerkiez_air_log` (unique on measurement `ts`; BLUME publishes hourly so half the ticks are dedup no-ops) → recompute that Berlin day's rollup in `schillerkiez_air_daily` → prune hourly rows > 90 d. Day bucketing is Europe/Berlin (`berlinDayKey`/`lastBerlinDays` in `src/lib/kiez/airLog.ts`, DST-safe via noon-UTC stepping). **Gaps are never interpolated**: a day without readings has NO rollup doc and is served as `lqiMax: null`.
- **`GET /api/kiez-air-history`**: public, Mongo-only (works while BLUME is silent) — exactly 7 day slots oldest→today + `lastReading` for the kiosk state §04 strip. `GET /api/kiez-air` (live) is unchanged; both share `src/lib/kiez/blume.ts`.
- **Reference import (Berlin-Vergleich)**: `syncReference()` in `scripts/sync-stats.ts` reads the MSS **Bezirke** XLSX (`MSS_BEZIRKE_XLSX_URL` — a different file from the PLR-level `MSS_XLSX_URL`) → `schillerkiez_reference` (`scope: berlin|neukoelln`, unique `{scope, period}`). Neukölln = Bezirk row `08`; Berlin = EW-weighted mean of the 12 Bezirke (no Berlin row in the file — approximation, recorded in `derivation`). `/api/kiez-stats` adds `reference?` only for the exact `latestSocial.period` (no back-filling); absent ⇒ field omitted ⇒ module quietly absent. Feeds `KzBerlinVergleich.svelte`'s dumbbell chart.
- **MSS URLs moved (July 2026)**: berlin.de relocated MSS files from `/sen/sbw/…` to `/sen/stadt/…` — the old `MSS_XLSX_URL`/`MSS_SDI_URL` values 404. Update `.env` + GitHub secrets when touching the sync.

### Kiosk frontend architecture (`/schillerkiez`, moss accent)
- **Page + mount**: `src/pages/schillerkiez.astro` → `KioskLayout` + `KiezPageInner.svelte` (`client:load`). Prerendered shell, hydrates client-side. Print route: `src/pages/schillerkiez/druck.astro` (A4 riso sheet, own QR-code render — see below).
- **Orchestrator + states contract**: `KiezPageInner.svelte` owns 4 independent `$state` fetches — `refetchStats()` (`/api/kiez-stats`, states `loading`/`ready`/`error`, retry button re-runs without a page reload), `fetchAir()` (`/api/kiez-air`, states `loading`/`ready`/`off` — §04), `fetchHistory()` (`/api/kiez-air-history`, best-effort, sparkline simply absent on failure), `fetchKontext()` (`/api/kiez-kontext`, best-effort, chips simply absent on failure). The air strip (`KzInstrumentStrip`) is mounted OUTSIDE the stats `{#if}` chain so a dead `/api/kiez-stats` never takes the strip down (and vice versa) — the two sources are deliberately decoupled. Derived states layered on top of `stats`: `isEmpty` (ready but no `demographics`), `isStale` (`lastUpdated` > 8 months old → §07 warn banner, page stays fully usable). State inventory maps to the design's `states.jsx`/`kiosk-kiezdaten-states.jsx` numbering (§01 loading, §02 stats error, §04 air off, §05 per-area social blank, §06 sparkline gap, §07 stale).
- **View-model lib**: `src/lib/kiez/kiezViewModel.ts` — pure, client-safe (`NO db/fetch imports`, enforced by a header comment; imported by both server `.astro` frontmatter and client Svelte islands). `buildKiezViewModel(stats)` maps the raw `KiezStatsResponse` into `KiezVM` (`areas: KzAreaVM[]`, `[0]` = Gesamt aggregate then the 4 PLRs in code order; `divTrend`; `socTrend`). `KzAreaVM.social` is `null` when a PLR has no MSS row for the current period — this is what drives the §05 honest-blank card, never a fake zero.
- **Wobble hand rules**: `src/lib/kiez/kzWobble.ts` — deterministic hand-drawn riso look for every SVG chart (`kzRnd(i, seed)` sine-based PRNG, same seed ⇒ same wobble ⇒ SSR/client markup matches). `kzWobLine`/`kzWobRect` are the two primitives every `Kz*` chart component calls into; never use raw straight SVG lines/rects for a data mark in this subtree.
- **Selector**: `KzSelector.svelte` — riso `KzMap` + a chip row (`Gesamt` + 4 PLRs) bound via `plr = $bindable`. The selector itself stays mounted across selections (own scroll/focus position persists); the Kanäle below re-key on `{#key plr}` in `KiezPageInner` so their draw-in animation (`kz-swap-in`) replays per selection. Mobile chip row uses the shared `kiosk-scroll-fade` + `scrollFade` action (see root `CLAUDE.md`'s "Horizontal scroll-fade utility") instead of the old carousel/scroll-snap pattern.
- **ZdW rotation**: `src/lib/kiez/zdw.ts` — pure, deterministic "Zahl der Woche" pick. ISO week number seeds a rotating start index into a fixed 5-item menu (`ageShare`, `populationDelta`, `diversity`, `singleHouseholds`, `airWeekMean`); each candidate is tried in rotation order and the first one whose data is actually derivable wins — an underivable figure is skipped, never faked. All-underivable ⇒ card hidden. Feeds the forum-share link in `KzTitleBlock.svelte` (prefills a `/topics/create` draft quoting the figure + page URL).
- **Kontext cache + asymmetry**: `src/lib/kiez/kontext.ts` — `getKiezKontext()`, 24h cache in `kiezKontextCache` (same in-code-TTL pattern as `chronikCache`). Curated `KEYWORDS` map (code-side, one regex per Kanal-04 row: `alq`/`kinderarmut`/`transfer`) matches against `topics.title`, threads must be > 1h old (spam guard), max 3 chips per row. **Deliberately asymmetric**: this lib reads the forum; the forum has no idea Kiez-Daten links to it (like the Kurier "heat" asymmetry elsewhere in the app). **Public gate is stricter than forum visibility**: approved-or-absent only (`{ $or: [{ moderationStatus: 'approved' }, { moderationStatus: { $exists: false } }] }`) — this deliberately EXCLUDES the forum's own pending+reported-stays-visible clause. A curation feature must never link under-review content, even content the requesting user could see in their own feed (there is no requesting user here — the chips are public). `beratung` (in the `transfer` keyword list) has known compound-word false-positive potential (e.g. "Rechtsberatung", "Eheberatung") — currently 0 real matches against live data so unobserved in practice; tune when real content arrives and false positives surface.
- **Druck route**: `src/pages/schillerkiez/druck.astro` — A4 print sheet, own server-side `qrSvg` render (`qrcode` package, `set:html`) encoding the page's public URL. `Astro.response.headers` sets `no-store`. QR fallback origin (when `getTrustedBaseUrl()` can't resolve one) is `https://mahalle-das-kiezgesichterbuch.vercel.app` (the current Vercel deploy origin) — **differs** from `steckbrief.astro`'s fallback of `https://mahalle.berlin`. Align both to the custom domain once it goes live; until then this is a known, deliberate asymmetry, not a bug.
- **Berlin-Vergleich dumbbell (`KzBerlinVergleich.svelte`)**: per-indicator (alq/ka/tr) dumbbell chart, Schillerkiez/Neukölln/Berlin dots on a **fixed 0–42% x-domain**, JSX-faithful, no clamp. Every real value comfortably fits today, but NK Kinderarmut (36.45%) already sits at ~87% of the cap — revisit the domain if a future MSS sync pushes a value past 42%.
- **Other fixed domains**: K02 (`KzKanalAge.svelte`) age bars use a fixed 42% domain and K04 (`KzKanalSocial.svelte`) social bars a fixed 50% domain — both fit real data today (max age bucket ~40%, max social ~37%); clamp/revisit if a future sync approaches the caps.
- **K05 shared-axis tradeoff**: `KzKanalSocTrend.svelte`'s Gesamt chart puts alq/ka/tr on one shared value axis (combined min/max across all three series) rather than a fixed pixel formula — unlike K01/K03/Berlin-Vergleich, the axis is data-driven because real pre-2021 child-poverty rates run up to ~76% (larger, poorer pre-reform Planungsräume), far outside any fixed mock domain. Known side-effect: alq/tr visually flatten against ka's larger range on the shared axis. This is inherited from the design's single-axis choice — per-metric normalization would deviate from the design source, so it was left as-is.
- **JSX handoff pointer**: design source lives at `design/handoffs/design_handoff_kiezdaten/jsx/` — `kiosk-kiezdaten.jsx` (desktop primary), `kiosk-kiezdaten-mobile.jsx`, `kiosk-kiezdaten-novel.jsx` (§02 Berlin-Vergleich / Anwohner-Kontext novel modules), `kiosk-kiezdaten-states.jsx` (state matrix). Every `Kz*` component's header comment cites the exact spec line range it ports.
